    - ParseLevelName("read") returns PermissionLevel.Read (4)
    - ParseLevelName("edit") returns PermissionLevel.Edit (20)
    - ParseLevelName("create") returns PermissionLevel.Create (84)
    - ParseLevelName("delete") returns PermissionLevel.Delete (340)
    - ParseLevelName("all") returns PermissionLevel.All (1364)
    - ParseLevelName("NONE") returns PermissionLevel.None (case-insensitive)
    - ParseLevelName("unknown") falls back to LevelValue cast
  </behavior>
  <action>
    1. Add a static method `ParseLevelName(string levelName)` to PermissionMapper that reverses GetLevelName:
       - Switch on levelName.ToLowerInvariant(): "none"->None, "read"->Read, "edit"->Edit, "create"->Create, "delete"->Delete, "all"->All
       - Default: throw ArgumentException (caller should use LevelValue as fallback)

    2. Add an instance method `ApplyPermissions(int pageId, List<SerializedPermission> permissions)` to PermissionMapper:
       - Per D-11: If permissions list is null or empty, return immediately (no changes to target)
       - Per D-02: Clear existing explicit permissions on the page first:
         - Use PermissionService.GetPermissionsByQuery to get existing permissions
         - Set each to PermissionLevel.None via SetPermission to clear (source-wins model)
       - Per D-04 + specifics: Build a group name->ID cache once: call UserManagementServices.UserGroups.GetGroups(), build Dictionary<string, int> keyed by group.Name (case-insensitive via StringComparer.OrdinalIgnoreCase)
       - Track whether any group was unresolvable (for safety fallback)
       - For each SerializedPermission:
         - Determine PermissionLevel: try ParseLevelName(perm.Level), if that throws use (PermissionLevel)perm.LevelValue as authoritative fallback
         - Create identifier: new PermissionEntityIdentifier(pageId.ToString(), "Page")
         - Per D-03: If perm.OwnerType == "role": call PermissionService.SetPermission(perm.Owner, identifier, level) and log per D-08: "Applied {Owner} = {Level} on page {pageId}"
         - Per D-04: If perm.OwnerType == "group": look up perm.Owner in group name cache (case-insensitive per specifics)
           - If found: call PermissionService.SetPermission(groupId.ToString(), identifier, level) and log per D-08
           - If not found: log per D-09: "Skipped permission for '{Owner}' -- group not found on target" and set unresolvable flag
       - Per D-05/D-06: After processing all permissions, if any group was unresolvable:
         - Call PermissionService.SetPermission("Anonymous", identifier, PermissionLevel.None)
         - Log per D-07/D-10: "Group '{name}' not found on target. Setting Anonymous=None on page {pageId} as safety fallback."

    3. Move the group cache building to a separate private method `BuildGroupNameCache()` that returns Dictionary<string, int>. Cache it as a field `_groupNameCache` (lazy, built on first call to ApplyPermissions) so it is reused across pages in the same deserialization run per specifics.

    4. Create tests/Dynamicweb.ContentSync.Tests/Serialization/PermissionDeserializationTests.cs:
       - Test ParseLevelName for all 6 known levels
       - Test ParseLevelName case-insensitivity
       - Test ParseLevelName throws ArgumentException for unknown string
