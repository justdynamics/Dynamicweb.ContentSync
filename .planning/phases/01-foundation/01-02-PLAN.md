    2. Create `src/Dynamicweb.ContentSync/Infrastructure/FileSystemStore.cs` in namespace `Dynamicweb.ContentSync.Infrastructure`:

       **Constructor:** Takes no constructor arguments. Creates ISerializer and IDeserializer from `YamlConfiguration.BuildSerializer()` and `YamlConfiguration.BuildDeserializer()`.

       **WriteTree(SerializedArea area, string rootDirectory):**
       - Create area directory: `Path.Combine(rootDirectory, SanitizeFolderName(area.Name))`
       - Write `area.yml` in the area directory — serialize the area record WITHOUT its Pages collection (create a copy with Pages = empty list, or use a YAML attribute to skip Pages)
       - Sort pages by `p.SortOrder` then `p.Name` (deterministic)
       - For each page, compute folder name via `GetPageFolderName()` (handles dedup)
       - Write `page.yml` in the page folder — serialize the page record WITHOUT its GridRows collection
       - Sort grid rows by `gr.SortOrder`
       - For each grid row, create folder `grid-row-{sortOrder}` inside page folder
       - Write `grid-row.yml` in the grid row folder — serialize the grid row WITHOUT its Columns collection (but include column metadata as inline data)
       - For each column, sort paragraphs by `p.SortOrder`
       - For each paragraph, write `paragraph-{sortOrder}.yml` in the grid row folder

       **CRITICAL: Pre-serialization dictionary key sorting.** Before serializing any object that has a `Dictionary<string, object> Fields` property, sort the dictionary keys alphabetically:
       ```csharp
       private Dictionary<string, object> SortFields(Dictionary<string, object> fields)
           => new(fields.OrderBy(kv => kv.Key));
       ```
       Apply this in the write path to the page and paragraph Fields before passing to the serializer.

       **ReadTree(string rootDirectory):**
       - Find the single area subdirectory in rootDirectory
       - Read `area.yml` and deserialize to SerializedArea
       - Find page subdirectories (those containing `page.yml`)
       - For each, read `page.yml` and deserialize
       - Find grid row subdirectories (those containing `grid-row.yml`)
       - For each, read `grid-row.yml`, then find `paragraph-*.yml` files
       - Reconstruct the full tree with children populated

       **SanitizeFolderName(string name):**
       - Replace characters from `Path.GetInvalidFileNameChars()` with underscore `_`
       - Trim leading/trailing whitespace
       - If result is empty, use `_unnamed`
       - Preserve spaces (per locked decision)

       **GetPageFolderName(string sanitizedName, Guid pageGuid, HashSet<string> usedNames):**
       - If `usedNames` does not contain `sanitizedName` (case-insensitive), add it and return `sanitizedName`
       - If duplicate, return `"{sanitizedName} [{pageGuid.ToString("N")[..6]}]"` and add to usedNames

       **Path length safety:**
       - Before writing any file, check if the full path exceeds 248 characters (directory) or 260 characters (full path)
       - If exceeded, truncate the folder name to fit and append `[{guid-prefix}]`
       - Log a warning to Console.Error (proper logging comes in Phase 5)
