4. **Test: Serialize_CustomerCenter_ProducesYamlTree** (SER-03 structure):
   - Discover the areaId at runtime: `var page = Services.Pages.GetPage(8385); var areaId = page.AreaId;`
   - Build a `SyncConfiguration` with `OutputDirectory = _outputDir`, one `PredicateDefinition` with `Name = "CustomerCenter"`, `AreaId = areaId`, `Path = "/" + page.MenuText`.
   - Create `ContentSerializer(config)` and call `Serialize()`.
   - Assert: `Directory.Exists(_outputDir)`.
   - Assert: `Directory.EnumerateFiles(_outputDir, "*.yml", SearchOption.AllDirectories).Any()` — at least one YAML file produced.
   - Assert: a `page.yml` file exists somewhere in the output tree.

5. **Test: Serialize_CustomerCenter_GuidOnly_NoNumericCrossRefs** (SER-03 GUID-only):
   - Same setup as above.
   - After serialization, read all `.yml` files.
   - For each file, assert the content does NOT contain `"8385"` as a standalone value (the known page ID should not appear as a value in the YAML).
   - Assert: all `page.yml` files contain `pageUniqueId:` with a GUID-formatted value (regex match `[0-9a-f]{8}-[0-9a-f]{4}-`).

6. **Test: Serialize_CustomerCenter_FieldFidelity** (INF-02):
   - Same setup.
   - After serialization, find any `page.yml` file that has a `fields:` section.
   - Read its content and assert it is valid YAML (deserializes without exception via YamlDotNet).
   - Spot-check: if any field value contains `<` (HTML), assert the YAML string preserves it (not escaped to `&lt;`).

7. **Test: Serialize_CustomerCenter_Idempotent** (determinism for SER-03):
   - Serialize to `_outputDir`.
   - Capture all file contents (path -> content dictionary).
   - Serialize again to same directory (overwrite).
   - Capture all file contents again.
   - Assert: file count is identical; each file has identical content (byte-for-byte match).

8. **Test: Serialize_CustomerCenter_HasChildPages** (recursive hierarchy):
   - After serialization, check that at least one page folder contains a subfolder with its own `page.yml` (child page detected).
