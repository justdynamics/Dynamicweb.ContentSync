**Method `BuildMergeCommand(Dictionary<string, object?> row, TableMetadata metadata)`** returns `CommandBuilder`:

1. Determine columns:
   - `keyColumns` = `metadata.KeyColumns`
   - `allColumns` = `metadata.AllColumns`
   - `insertColumns` = all columns (including identity if identity is a key column)
   - `updateColumns` = all columns EXCEPT key columns (only non-key columns are updated on match)
2. Determine identity insert: `var enableIdentityInsert = metadata.IdentityColumns.Any(ic => metadata.KeyColumns.Contains(ic, StringComparer.OrdinalIgnoreCase))`
3. Build CommandBuilder:
   ```
   SET IDENTITY_INSERT [{table}] ON;  -- only if enableIdentityInsert
   MERGE [{table}] AS target
   USING (SELECT {param0}, {param1}, ...) AS source ([Col1], [Col2], ...)
   ON (target.[KeyCol] = source.[KeyCol])
   WHEN MATCHED THEN UPDATE SET [Col1] = source.[Col1], [Col2] = source.[Col2], ...
   WHEN NOT MATCHED THEN INSERT ([Col1], [Col2], ...) VALUES (source.[Col1], source.[Col2], ...);
   SET IDENTITY_INSERT [{table}] OFF;  -- only if enableIdentityInsert
   ```
4. For each column value: use `cb.Add("{0}", row[col] ?? DBNull.Value)` — CommandBuilder parameterizes via `{0}` placeholder. Map null Dictionary values back to `DBNull.Value` for SQL parameters.
5. The USING SELECT should have one `{0}` placeholder per column, each added via separate `cb.Add("{0}", value)` call with comma separators between them.

**Method `WriteRow(Dictionary<string, object?> row, TableMetadata metadata, bool isDryRun)`** returns `WriteOutcome` (enum: `Created`, `Updated`, `Skipped`, `Failed`):

1. Build MERGE command via `BuildMergeCommand`
2. If `isDryRun`: log the MERGE SQL (for debugging) but do NOT call `_sqlExecutor.ExecuteNonQuery`. Return `WriteOutcome.Created` or `WriteOutcome.Updated` as appropriate (check if row exists first).
3. If not dry-run: call `_sqlExecutor.ExecuteNonQuery(cb)`
4. Use `OUTPUT $action` in the MERGE to determine if it was an INSERT or UPDATE:
   Modify MERGE to include `OUTPUT $action` and use `ExecuteReader` instead of `ExecuteNonQuery` to read the action. Or simpler: query existence first with `SELECT COUNT(*) FROM [{table}] WHERE {keyMatch}`, then return Created or Updated based on pre-existence.

   Actually, simplest approach matching DW pattern: use `MERGE ... OUTPUT $action;` and read the output. But since CommandBuilder may not support mixing OUTPUT with ExecuteNonQuery cleanly, use the two-step approach:
   - Before MERGE: check existence via `SELECT 1 FROM [{table}] WHERE {keyCondition}` using `ExecuteReader`
   - Execute MERGE via `ExecuteNonQuery`
   - Return `Created` if row didn't exist before, `Updated` if it did

5. Wrap in try/catch — on exception, log error and return `WriteOutcome.Failed`

**Enum `WriteOutcome`** in same file or separate: `Created`, `Updated`, `Skipped`, `Failed`

**Method `RowExistsInTarget(TableMetadata metadata, Dictionary<string, object?> row)`** returns `bool`:
- Build `SELECT 1 FROM [{table}] WHERE [key1] = {0} AND [key2] = {1}`
- Use `ExecuteReader`, return `reader.Read()` result

For dry-run mode (per D-06 and Pitfall 6): When `isDryRun` is true, still check existence to report accurate Created vs Updated counts, but never execute the MERGE.
