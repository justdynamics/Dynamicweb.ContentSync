  <behavior>
    - Exact path match: ContentPredicate for path="/Customer Center", areaId=1 — ShouldInclude("/Customer Center", 1) returns true
    - Child path match: ShouldInclude("/Customer Center/Products", 1) returns true (starts with predicate path)
    - Unrelated path: ShouldInclude("/Blog", 1) returns false
    - Wrong areaId: ShouldInclude("/Customer Center", 2) returns false (areaId mismatch)
    - Exclude overrides include: predicate path="/Customer Center", excludes=["/Customer Center/Archive"] — ShouldInclude("/Customer Center/Archive", 1) returns false
    - Exclude child: ShouldInclude("/Customer Center/Archive/Old", 1) returns false (child of excluded path)
    - Non-excluded sibling: ShouldInclude("/Customer Center/Products", 1) returns true (not under any exclude)
    - Multiple excludes: predicate with excludes=["/Customer Center/Archive", "/Customer Center/Drafts"] — both excluded paths return false, other children return true
    - Case-insensitive path matching: ShouldInclude("/customer center/Products", 1) returns true
    - Path boundary: ShouldInclude("/Customer Center2", 1) returns false (not a child — different path segment)
    - Aggregate evaluator: ContentPredicateSet constructed from SyncConfiguration with multiple predicates — ShouldInclude returns true if ANY predicate includes the path (logical OR)
  </behavior>
  <action>
    **ContentPredicate.cs** — class in `Dynamicweb.ContentSync.Configuration` namespace:

    Single-predicate evaluator:
    - Constructor: `ContentPredicate(PredicateDefinition definition)`
    - `bool ShouldInclude(string contentPath, int areaId)`:
      1. If `areaId != definition.AreaId` return false
      2. Normalize: ensure contentPath and definition.Path comparison is case-insensitive (use `OrdinalIgnoreCase`)
      3. Check include: contentPath equals definition.Path OR contentPath starts with definition.Path + "/" (path boundary check — prevents "/Customer Center2" matching "/Customer Center")
      4. Check exclude: if any exclude path in definition.Excludes matches (contentPath equals exclude OR starts with exclude + "/"), return false
      5. Return include result

    Aggregate evaluator (inner class or separate):
    - `ContentPredicateSet` class with constructor taking `SyncConfiguration`
    - Creates a `ContentPredicate` for each `PredicateDefinition`
    - `bool ShouldInclude(string contentPath, int areaId)` — returns true if ANY predicate includes the path

    **ContentPredicateTests.cs** — xunit test class covering all behaviors above. Use `[Theory]` with `[InlineData]` for parameterized path/expected-result tests where possible.
  </action>
